Manual de Uso - APDC Herramienta de conversion del reporte Check Positive Pay extraído de Quickbooks en el formato requerido por la plataforma de TD Bank para la verificación de transacciones y actualizacion del archivo con el historial de transacciones verificadas

Descripción

Esta aplicación está construida en Python y Tkinter con el objetivo de automatizar el proceso de la creacion de un archivo de tipo CSV desde el reporte de Quickbooks Check Positive Pay con el formato y datos especificos requeridos por la plataforma de TD Bank para la verificacion de transacciones, a su vez tambien actualiza el archivo tipo XLSX que se utiliza para llevar el registro historico de todas las transacciones verificadas.

Requisitos del sistema

  • Para la ejecución del programa no se requiere ningún requerimiento de software específico. Sin embargo, para un funcionamiento correcto y obtener los resultados esperados, se necesitan las siguientes condiciones:
  • Reporte "Check Positive Pay" exportado desde Quickbooks
    • Los datos del reporte deben estar en una hoja llamada "Sheet1". Por lo general, el archivo se exporta con esta configuración predeterminada.
    • El archivo debe ser de tipo .XLSX, es decir, una hoja de cálculo de Excel en la versión 2007 o posterior.
  • Archivo con registro historico de todas las transacciones verificadas
    • Este archivo registra todas las transacciones verificadas por cada una de las cuentas separadas cada una en una pagina diferente.
    • Es importante tener en cuenta el formato utilizado por el software para detectar correctamente los datos a extraer y modificar. Cada una de las paginas debe de tener unica y exclusivamente su nombre con los ultimos cuatro digitos de la cuenta en cuestion, por ejemplo, 2674. Asi mismo se deben de mantener las columnas de los datos en el orden siguiente, fecha, numero de cuenta completo, numero de transaccion, nombre de la persona a quien va dirigida la transaccion, monto de la transaccion.
    • Es necesario que este archivo se mantenga cerrado cuando se este ejecutando el procesamiento de los datos ya que el software realiza operaciones de lectura y escritura en el mismo.

Instalación

  • No es necesario realizar ninguna instalación.

Ejecución

  • Hacer doble clic en el archivo ejecutable proporcionado.

Funcionalidades principales

Selección de archivos

  • En la interfaz gráfica, se muestran dos secciones: "Verified Transaction History File" y "Check Positive Pay Quickbooks Report".
  • En la sección "Check Positive Pay Quickbooks Report", se puede seleccionar un único archivo de tipo XLSX (Excel versión 2007 o posterior) haciendo clic en el botón "Select file". Aquí se debe seleccionar el reporte de tipo "Check Positive Pay" exportado de Quickbooks.
  • En la sección "Verified Transaction History File", se puede seleccionar un único archivo de tipo XLSX (Excel versión 2007 o posterior) haciendo clic en el botón "Select file". Aquí se seleccionará el archivo con el registro historico de todas las transacciones verificadas.

Procesamiento de archivos

El procesamiento de archivos se lleva a cabo siguiendo los pasos descritos a continuación:

Selección de archivos

  • Seleccione los archivos necesarios y luego haga clic en el botón "Start Process" para iniciar el procesamiento.

Progreso y mensajes

  • Durante el procesamiento, se mostrará una barra de progreso que proporciona información general sobre el proceso en curso.
  • En la parte inferior de la interfaz, si es el caso se mostrará una caja de texto con mensajes relevantes sobre el proceso.

Situaciones durante el procesamiento

  1. Detección de transacciones nulas o vacías: Es posible que el reporte de Quickbooks contenga transacciones sin monto especificado.

Procesamiento de los datos

A continuación se menciona el ciclo que ejecuta el software para el procesamiento de los datos.

  1. Lectura de los archivos seleccionados.
  2. Extraccion de información de reporte de Quickbooks Check Positive Pay
  3. Creacion de grupos independientes de datos por cada cuenta detectada
  4. Extraccion de información de reporte de archivo con registro historico de todas las transacciones verificadas
  5. Creacion de grupos independientes de datos por cada cuenta detectada
  6. Comparacion y busqueda de coincidencias entre datos extraidos
  7. Identificacion de nuevas transacciones
  8. Tratamiento de transacciones nulas o vacias
  9. Preparacion de datos con formato especifico para plataforma TD Bank
  10. Exportacion de archivos de tipo CSV y XLSX para plataforma TD Bank
  11. Preparacion de datos con formato especifico para archivo con registro historico de todas las transacciones verificadas
  12. Insercion de datos en archivo con registro historico de todas las transacciones verificadas
  13. Guardado de archivo con registro historico de todas las transacciones verificadas

Resultados

El software genera los siguientes resultados:

Cada uno de los archivos resultantes tendrá por nombre una estructura de tiempo que consiste en la palabra "APDC" junto al mes, dia, año y hora en la cual fue realizado el proceso, por ejemplo, APDC 07052023 AM11

Estos archivos se van a ubicar en la raiz del archivo ejecutable:

  • Archivo de tipo XLSX: Este archivo contiene los datos y el formato especifico requerido por la plataforma de TD Bank para la verificacion de transacciones. Este archivo solo se exporta para fines de verificacion y de manejo de los datos pero no debe ser subido a la plataforma bancaria.
  • Archivo de tipo CSV: Este archivo contiene los datos y el formato especifico requerido por la plataforma de TD Bank para la verificacion de transacciones. Este archivo debe ser subido a la plataforma bancaria.

Nota

  • Cada vez que se ejecute el procesamiento de datos, todos los archivos resultantes de este nuevo proceso van a sobrescribir los archivos creados en procesos anteriores, siempre y cuando se ejecute el proceso el mismo dia y a la misma hora (nombres identicos).

Archivo con registro historico de todas las transacciones verificadas

  • Este archivo se actualizará con los resultados obtenidos, insertando los datos correspondientes en las hojas designadas.

Notas adicionales

  • Asegúrese de tener los archivos necesarios en el formato correcto antes de comenzar el procesamiento.
  • Si se experimenta algún problema o error, verifique que los archivos seleccionados sean válidos y cumplan con los requisitos esperados.
