User Manual - APDC Conversion tool of the check positive pay report from Quickbooks to a TD Bank platform required formta for transaction verification and file updating with the verified transactions record

Description

This application is built in Python and Tkinter with the aim of automating the process of creating a CSV file from the Quickbooks Check Positive Pay report with the specific format and data required by the TD Bank platform for transaction verification. It also updates the XLSX file used to keep a historical record of all verified transactions.

System Requirements

  • No specific software requirements are needed for running the program. However, to ensure proper functioning and expected results, the following conditions are required:
  • "Check Positive Pay" report exported from Quickbooks
    • The report data should be in a sheet named "Sheet1". Typically, the file is exported with this default configuration.
    • The file must be in .XLSX format, i.e., an Excel spreadsheet in version 2007 or later.
  • File with a record of all verified transactions
    • This file records all the verified transactions for each separate account on different pages.
    • It is important to note the format used by the software to correctly identify the data to extract and modify. Each page must have its unique name consisting of the last four digits of the respective account, for example, 2674. The data columns should be maintained in the following order: date, complete account number, transaction number, recipient's name, transaction amount.
    • This file must remain closed when processing the data, as the software performs read and write operations on it.

Installation

  • No installation is required.

Execution

  • Double-click on the provided executable file.

Main Functionality

File Selection

  • In the graphical interface, two sections are displayed: "Verified Transaction Record File" and "Check Positive Pay Quickbooks Report".
  • In the "Check Positive Pay Quickbooks Report" section, you can select a single XLSX file (Excel version 2007 or later) by clicking the "Select file" button. Here, you should select the "Check Positive Pay" report exported from Quickbooks.
  • In the "Verified Transaction Record File" section, you can select a single XLSX file (Excel version 2007 or later) by clicking the "Select file" button. Here, you should select the file with the record of all verified transactions.

File Processing

The file processing is carried out following the steps described below:

File Selection

  • Select the required files and then click the "Start Process" button to initiate the processing.

Progress and Messages

  • During processing, a progress bar will be displayed, providing general information about the ongoing process.
  • At the bottom of the interface, relevant messages about the process may be displayed in a text box, if applicable.

Situations during Processing

  1. Detection of null or empty transactions: It is possible that the Quickbooks report contains transactions without specified amounts.

Data Processing

Below is the cycle executed by the software for data processing:

  1. Reading the selected files.
  2. Extraction of information from Quickbooks Check Positive Pay report.
  3. Creation of independent data groups for each detected account.
  4. Extraction of information from the file with a record of all verified transactions.
  5. Creation of independent data groups for each detected account.
  6. Comparison and search for matches between extracted data.
  7. Identification of new transactions.
  8. Handling of null or empty transactions.
  9. Preparation of data in the specific format for the TD Bank platform.
  10. Export of CSV and XLSX files for the TD Bank platform.
  11. Preparation of data in the specific format for the file with a record of all verified transactions.
  12. Insertion of data into the file with a record of all verified transactions.
  13. Saving the file with a record of all verified transactions.

Results

The software generates the following results:

Each of the resulting files will be named with a time structure consisting of the word "APDC" followed by the month, day, year, and hour when the process was performed. For example, APDC 07052023 AM11

These files will be located in the root directory of the executable file:

  • XLSX File: This file contains the data in the specific format required by the TD Bank platform for transaction verification. This file is only exported for verification purposes and data management but should not be uploaded to the banking platform.
  • CSV File: This file contains the data in the specific format required by the TD Bank platform for transaction verification. This file should be uploaded to the banking platform.

Note

  • Each time the data processing is executed, all resulting files from this new process will overwrite the files created in previous processes, as long as the process is executed on the same day and at the same time (identical names).

File with a Record of All Verified Transactions

  • This file will be updated with the obtained results, inserting the corresponding data into the designated sheets.

Additional Notes

  • Make sure to have the necessary files in the correct format before starting the processing.
  • If any issues or errors are encountered, verify that the selected files are valid and meet the expected requirements.
